jQuery: Administración de eventos

jQuery facilita la administración de eventos de javascript y lo más importante hace transparente la diferencia en la registración de eventos entre distintos navegadores (IExplorer, FireFox, Chrome, Safari, Opera)

$(documento).ready(nombre de función)

Se Dijo que este función que se registra mediante el método ready() se ejecuta cuando el DOM del documento está en memoria. Si no se utiliza la librería jQuery se hace esto a través del evento addEventListener .

Otro evento que se ha visto en ejemplos anteriores el evento click de un elemento, la sintaxis utilizada: 

            let x;
            x=$("button");
            x.click(() => body function;);

Con este pequeño código se registra la función anónima para todos los elementos de tipo button del documento.

Con este otro código:

            let x;
            x=$("#boton1");
            x.click(() => body function;);

solo se registra el evento click para el elemento con id con valor #boton1

Por último con este código se registra para todos los elementos "button" que dependen del div con valor de id #formulario1:

            var x;
            x=$("#formulario1 button");
            x.click( () => body function;);

Ejercicio Manejo de eventos

Para reafirmar estos conceptos se creará una página que muestre un párrafo, luego una tabla que contenga dos párrafos y por último otra tabla que contenga dos párrafos. Se capturá el evento click de todos los párrafos del documento y se mostrará un mensaje indicando:'se presionó un párrafo del documento', también se capturará el evento click de los dos párrafos de la segunda tabla y se mostrará un mensaje "se presionó un párrafo contenido en la segunda tabla"
